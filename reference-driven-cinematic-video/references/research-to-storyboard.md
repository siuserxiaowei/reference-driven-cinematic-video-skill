# Research To Storyboard

Use this when product information is incomplete or when the user asks Codex to search and think.

## Research Pass

Search for:

- official product/site/docs/repo
- public demos, screenshots, app store listings, launch posts
- competitors and category language
- visual references from similar launches
- audience pain language from forums, reviews, or public comments when appropriate

Keep notes short:

```markdown
Source:
Useful facts:
Visual clues:
Claims safe to use:
Claims not safe to use:
```

## Strategy Pass

Turn research into a video argument:

1. What does the audience already believe?
2. What problem must the first five seconds make obvious?
3. What is the product's simplest promise?
4. What proof can be shown instead of said?
5. What should the viewer remember after one watch?

## Reference Board

When no single reference video is supplied, search for 3-6 public examples or projects and choose one primary direction. Keep the board short:

```markdown
Reference:
Useful visual grammar:
Useful workflow idea:
Do not copy:
How it maps to the product:
```

Good reference sources include official product launch videos, public demo clips, GitHub video-as-code projects, Remotion/HyperFrames/Motion Canvas examples, and credible competitor explainers.

```markdown

| 14-24s | Proof/workflow | | | | | |
| 24-34s | Outcome | | | | | |
| 34-40s | Close | | | | | |
```

## Copy Rules

- On-screen text should be shorter than voice.
- Voice explains; image proves.
- Avoid "not only... but also..." and generic AI/product hype.
- If a line sounds like a brochure, rewrite it as something a founder could say out loud.
- Use one visual idea per shot.
- Every claim should be visualized or softened. If there is no proof, make it positioning language instead of a factual promise.
